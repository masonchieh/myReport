💡重要注意事項

⚠️務必安裝 Markdown All in One extensions

🛠️ 1. ./vscode/settings.json 此設定檔可依據個人習慣調整

  • 這個設定檔是調整 Markdown All in One
    1. "markdown.extension.print.onFileSave": false ：如果設定 true 會自動產生相對應的 .html，但現在 手動統一產生在projRootPath/out，所以預設為 false，避免混淆，此行為類似 滑鼠按右鍵，選擇 Print current document to HTML
    2. Print documents to HTML (select a source folder)：產生整包 project 的 .html 沒有設定值可以設定，所以只能 手動執行

🚀 2. 如何手動 產生整包 project 的 .html ?

  • 在任一 .md 檔案上，按滑鼠右鍵，點選 Print documents to HTML (select a source folder)
  • 選擇 projRootPath
  • vscode 上方會彈出對話視窗，預設路徑為 projRootPath/out 按下 Enter
  • 會自動產生所有 .md 檔案相對應的 .html 檔
  • 完成 project 的 /out/ html 內容輸出

⌘ 3. Ctrl + Shift + b

  • 執行 task 使得 /out/ 方便攜帶
    1. /images/ 整包複製至 /out/ 下
    2. 修正 .html 檔案內的 絕對路徑，變成 相對路徑，因 Print documents to HTML (select a source folder) 產生 絕對路徑，不利於 /out/ 攜帶，會受限於 本機圖檔資源 存取
    3. 改為 相對路徑 並同步 圖檔資源
