SQLite local db

More...
  • 目的：app 本地端儲存資料，使用 file(data.db) 儲存進行 i/o 行為
  • 敏感資料有進行 加解密
  • 檔案可隨意存取，使用 Viewer ( SQLiteStudio 或 DB.Browser.for.SQLite ) 即可開啟瀏覽 data，因為是 open source
  • 免費原生 SQLite 無法在開啟時，要求輸入帳密，須改用商業版 SQLCipher、SQLite SEE 才可以鎖住 data.db 檔案的存取
  • 使用 免費原生 SQLite 又想要保密，只能 自行從應用層，將敏感資料進行 加解密
  • 存取 data 時，可搭配 AES encrypt/decrypt 進行資料的保護
  • 最好搭配 ORM(Object-Relational Mapping) tool 進行資料存取
    • rusqlite、sqlx 類似於 dapper 的操作
    • diesel、sea-orm 類似於 entity-framework 的操作
    • 因為 rusqlite 只支援同步，沒有支援異步操作，所以換成 sqlx 實作
